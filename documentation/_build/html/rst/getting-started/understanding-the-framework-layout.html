1.2.2. Folder: firmware¶

This folder is meant to hold the firmware applications you make. The default applications are:

  • HelloWorld: Template for simple single-threaded applications.

1.2.3. Folder: firmware/<application>/source/¶

The source folder holds the main.cpp file and other source files you want to create or use for your project.

1.2.4. Folder: <applications>/build/binaries/¶

This folder holds the executables (.hex) that can be loaded into the SJTwo board.

It also holds the disassembly file .lst, linker file .map and the Executable and Linkable Format .elf file.

1.2.5. Folder: <application>/build/_can_dbc/¶

The _can_dbc folder holds the CAN message description files and generated header files.

1.2.6. Folder: <application>/build/compiled/¶

This folder holds object files generated during the compilation stage of building the executable. They are then all linked together to create an .elf file afterwards. The last phase after creating an .elf is to converts the .elf to a .hex file to be loaded into SJTwo’s flash memory.

1.2.7. Folder: <application>/build/test/¶

This folder holds compiled test object files as well as the test.exe executable file. Running the test.exe file will run through all of the library tests as well as any local test files in your project.

1.2.8. Folder: <application>/build/coverage/¶

This folder holds the test coverage files. Opening the coverage.html in your browser will show you how much of the library code has been tested.

1.2.9. Folder: firmware/library/¶

This folder holds the core SJSU-Dev2 firmware source files, such as abstractions for using GPIO, I2C, UART, Interrupts, etc.

1.2.10. Folder: firmware/third_party¶

This folder holds the third party projects that are used by the SJSU-Dev2 framework.

1.2.11. Folder: firmware/lib/L%d_%s¶

The folders that start with L<some number>_<some folder name> are kind of self explanatory as to what they hold. For example, L0_LowLevel holds files that initialize the firmware before main is called and L1_Drivers are device drivers and so on and so forth.
